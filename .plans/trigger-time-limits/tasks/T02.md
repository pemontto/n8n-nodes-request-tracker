## Goal
Inject a time constraint into all queries to prevent unbounded lookback. Minimum = poll interval.

## Context
Users may write queries without time filters. Even with result limits, the query itself can be slow on large RT instances. A backstop ensures queries only look at recent data.

## Requirements
- In polling mode: inject `${triggerOnField} > ${now - backstopDuration}`
- In manual mode: same backstop applies
- Backstop = max(poll interval, configurable default like 24h)
- User can override with larger value if needed
